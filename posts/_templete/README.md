# 記事テンプレート

新しい記事を書くときは、このフォルダをコピーして使う。

## 使い方

```bash
cp -r posts/_template posts/<記事のスラッグ>
```

例：

```bash
cp -r posts/_template posts/wget-vs-curl
```

スラッグは英小文字 + ハイフン区切り。`articles/<記事のスラッグ>.md` のファイル名と一致させる（命名規則で記事と作業フォルダを結びつけるため）。

## フォルダ構成と役割

``` txt
<記事のスラッグ>/
├── README.md     # この記事の調査メモ・参考リンク・TODO
├── draft.md      # 記事本文の下書き
└── snippets/     # 記事内で使うコードの検証用
```

### README.md

この記事専用の作業ノート。以下を書く想定：

- **この記事で書きたいこと** — 記事の方向性・主題・想定読者
- **参考リンク** — 調査中に見つけた一次情報・関連記事
- **サンプルコードの説明** — `snippets/` に何を置いたか、どう動かすか
- **TODO** — 残作業のチェックリスト

公開記事には載らない、自分のための索引として使う。

### draft.md

公開する記事本文の下書き。Zenn の Front Matter 付き。

完成したら `articles/<記事のスラッグ>.md` にコピーして公開する：

```bash
cp posts/wget-vs-curl/draft.md articles/wget-vs-curl.md
```

### snippets/

記事中で「こう書くとこう動く」を示すコードの検証用フォルダ。記事に貼るコードは、ここで実際に動かして確認したものを使う。

サブフォルダを切る場合の例：

``` text
snippets/
├── getopt-demo/
│   ├── main.c
│   └── Makefile
└── compare.sh
```

draft.md の末尾に `snippets/` への GitHub リンクを貼って、読者がコードを実際に動かせるようにする。

## このテンプレート自体の注意点

- `_template` というフォルダ名のアンダースコアは「実際の記事ではない」ことを示すための慣習。コピーした先のフォルダ名にはアンダースコアを付けない。
- `snippets/` はテンプレート段階では空。コピー先で必要に応じてファイルを追加する。
